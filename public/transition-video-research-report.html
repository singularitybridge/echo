First-Last Frame Accuracy Analysis

Research Report Fal.ai Models January 2025

Executive Summary

Current Implementation Has No End Frame Guarantee

The current implementation uses fal-ai/veo3.1/fast/first-last-frame-to-video which has NO documented end frame accuracy guarantees. Research reveals that dedicated keyframe interpolation models provide significantly better endpoint precision.

The Problem

Current transition scenes don't accurately end at the target end frame (start frame of next shot). The expected flow should be:

Shot 1 Last Frame
Reference Start
TRANSITION VIDEO
Should interpolate smoothly
Shot 3 First Frame
Target End
Issue: The transition video doesn't match the end frame precisely, causing visual discontinuity.

Detailed Findings

1. Current Model: Veo 3.1 Fast

  • No specific end frame accuracy documentation
  • Known issues: visual drift, temporal inconsistency
  • Motion prediction ~35% improved but "rapid panning still introduces artifacts"
  • Prompt length sweet spot: 150-300 characters

2. WAN 2.1/2.2 FLF2V RECOMMENDED

  • 98% first/last frame matching rate - explicitly designed for this use case
  • 37% reduced video jitter compared to similar models
  • Uses CLIP semantic features and cross-attention mechanisms
  • Bidirectional conditioning: receives guidance from both temporal directions
  • 14B parameter model trained specifically for frame interpolation
  • Flat pricing: $0.40 per 720p generation (not per second!)

3. Kling O1

  • "Dual-keyframe approach" specifically for endpoint precision
  • "Trades generation flexibility for endpoint accuracy"
  • Quality sweet spot is 3-5 seconds
  • Beyond 5 seconds, "temporal consistency degrades"
  • Supports @Image1 and @Image2 prompt references

4. Framepack FLF2V

  • Has explicit strength parameter (0-1) controlling end frame influence
  • Higher values = output more heavily influenced by last frame
  • Default 0.8 provides balance between prompt and destination frame
  • Very cheap: $0.0333/sec

Recommendations

Option A Recommended

Switch to WAN 2.1 FLF2V

  • Best documented end frame accuracy (98%)
  • Purpose-built for keyframe interpolation
  • Cost-effective flat rate pricing
  • API: fal-ai/wan-flf2v
Option B

Try Kling O1

  • Designed for "deterministic video outputs"
  • Good for narrative scene transitions
  • Slightly more expensive but quality-focused
  • API: fal-ai/kling-video/o1/image-to-video
Option C

Use Framepack with High Strength

  • Set strength: 0.95 or higher for maximum end frame fidelity
  • Very cost-effective
  • API: fal-ai/framepack/flf2v
